01. FastAPI lifespan hoạt động như thế nào? Tại sao không dùng on_event?

Lifespan là hàm async generator chia làm 2 phần: code trước yield chạy khi startup, code sau yield chạy khi shutdown. Đây là nơi khởi tạo DB pool, Redis, load ML model — những thứ tốn chi phí và nên tái sử dụng.

on_event đã deprecated từ v0.93. Lifespan tốt hơn vì: nếu startup lỗi giữa chừng, phần cleanup sau yield vẫn đảm bảo chạy.

from contextlib import asynccontextmanager
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    # startup -- chay 1 lan
    app.state.db    = await create_db_pool()
    app.state.redis = await create_redis()
    yield                          # app dang nhan request
    # shutdown -- chay 1 lan khi tat
    await app.state.db.close()
    await app.state.redis.close()

app = FastAPI(lifespan=lifespan)
Như ca trực bảo vệ: bật hệ thống khi vào ca, tắt và khóa cửa khi tan ca.
02. Dependency Injection trong FastAPI là gì? Cách hoạt động?

Dependency Injection là cơ chế FastAPI tự động gọi hàm phụ thuộc và truyền kết quả vào endpoint — thay vì endpoint phải tự khởi tạo DB, xác thực token. Dùng Depends() để khai báo.

Nếu nhiều endpoint trong cùng request cùng cần 1 phụ thuộc, FastAPI chỉ gọi 1 lần và tái sử dụng.

from fastapi import Depends

async def get_db():
    db = SessionLocal()
    try:
        yield db          # tra DB cho endpoint dung
    finally:
        db.close()        # dam bao dong du co loi

async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db = Depends(get_db),
):
    user = verify_token(token, db)
    if not user:
        raise HTTPException(status_code=401)
    return user

@app.get("/me")
async def me(user = Depends(get_current_user)):
    return user   # FastAPI tu inject user
Như hệ thống cấp phát đồ mổ trong bệnh viện: bác sĩ không tự đi lấy, có hệ thống chuẩn bị sẵn và thu hồi sau ca.
03. Middleware là gì? Thứ tự thực thi thế nào?

Middleware là code chạy với mọi request trước khi vào endpoint và mọi response trước khi trả về client. Dùng cho: logging, request ID, auth check, CORS, rate limiting.

Thứ tự quan trọng: middleware thêm sau thì chạy trước — như lớp vỏ hành từ ngoài vào trong.

@app.middleware("http")
async def add_request_id(request: Request, call_next):
    request_id = str(uuid4())
    request.state.id = request_id
    response = await call_next(request)   # goi middleware tiep / endpoint
    response.headers["X-Request-ID"] = request_id
    return response

# Thu tu thuc thi khi request den:
# LogMiddleware -> AuthMiddleware -> endpoint -> AuthMiddleware -> LogMiddleware
app.add_middleware(AuthMiddleware)   # them truoc -> chay sau (lop trong)
app.add_middleware(LogMiddleware)    # them sau  -> chay truoc (lop ngoai)
Như các trạm kiểm soát ở sân bay: check-in → soi chiếu → cửa ra, và ngược lại khi về.
04. Phân biệt HTTPException, custom exception handler và RequestValidationError?

Ba cách xử lý lỗi, dùng đúng chỗ:

HTTPException: lỗi HTTP đơn giản, chủ động throw trong code. Custom handler: lỗi business logic riêng, cần format thống nhất. RequestValidationError: Pydantic tự throw khi input sai kiểu — có thể ghi đè handler mặc định.

# 1. HTTPException -- loi HTTP chuan
@app.get("/users/{id}")
async def get_user(id: int, db=Depends(get_db)):
    user = db.get(User, id)
    if not user:
        raise HTTPException(status_code=404, detail="User not found")

# 2. Custom exception -- loi business logic co them data
class InsufficientFundsError(Exception):
    def __init__(self, balance: float, required: float):
        self.balance = balance
        self.required = required

@app.exception_handler(InsufficientFundsError)
async def handle_funds(request, exc: InsufficientFundsError):
    return JSONResponse(status_code=400, content={
        "error": "insufficient_funds",
        "balance": exc.balance,
        "required": exc.required,
    })

# 3. RequestValidationError -- ghi de format loi Pydantic
from fastapi.exceptions import RequestValidationError

@app.exception_handler(RequestValidationError)
async def validation_handler(request, exc):
    return JSONResponse(status_code=422, content={"detail": exc.errors()})
05. Background Tasks là gì? Khi nào dùng Celery thay vì Background Tasks?

Background Tasks: chạy task sau khi response đã trả về client, built-in không cần cài thêm. Giới hạn: nếu app restart thì task mất, không có retry, không monitor được.

Celery: hàng đợi task bền vững, cần Redis/RabbitMQ. Task được lưu, tự retry khi lỗi, có Flower để monitor.

from fastapi import BackgroundTasks

async def send_notification(email: str, data: dict):
    await email_client.send(email, data)

@app.post("/transfer")
async def transfer(req: TransferRequest, bg: BackgroundTasks):
    result = await process_transfer(req)
    bg.add_task(send_notification, req.email, result)  # khong block response
    return {"status": "success"}

06. Hàm async và hàm sync trong FastAPI — khi nào dùng cái nào?

Dùng async def khi gọi các hàm async (DB async, HTTP async). FastAPI chạy trong event loop — không block các request khác.

Dùng def thuần khi dùng thư viện đồng bộ. FastAPI tự chạy trong thread pool riêng — không block event loop.

Lỗi nguy hiểm: viết async def nhưng bên trong gọi hàm blocking — đóng băng toàn bộ event loop, không request nào khác được xử lý.

# Dung: async + thu vien async
@app.get("/users")
async def get_users(db: AsyncSession = Depends(get_async_db)):
    result = await db.execute(select(User))
    return result.scalars().all()

# Dung: sync + thu vien dong bo (FastAPI tu chay trong thread)
@app.get("/report")
def generate_report():
    return heavy_pandas_computation()   # blocking OK vi la def, khong async

# SAI: async + blocking call -> dong bang event loop
@app.get("/bad")
async def bad():
    time.sleep(5)          # BLOCK toan bo app!
    requests.get(url)      # BLOCK! -> dung httpx.AsyncClient thay
Le tan async giỏi: trong khi chờ phòng dọn, tiếp tục check-in khách khác. Le tan sync: thuê người phụ riêng ngồi phòng khác, không làm gián đoạn quầy chính.
07. FastAPI dùng ASGI hay WSGI? Khác nhau thế nào?

WSGI (Flask, Django cũ): đồng bộ, 1 request tại 1 thời điểm, không hỗ trợ WebSocket.

ASGI (FastAPI, Django 4+): bất đồng bộ, nhiều request đồng thời, hỗ trợ WebSocket và Server-Sent Events. FastAPI xây trên ASGI qua Starlette, chạy bằng Uvicorn.

# Development
uvicorn main:app --reload

# Production: nhieu process Uvicorn qua Gunicorn -> da nhan CPU + async
gunicorn -w 4 -k uvicorn.workers.UvicornWorker main:app

# Hoac chay nhieu instance qua Docker + load balancer
WSGI như đường 1 làn — xe nối đuôi nhau. ASGI như đường cao tốc nhiều làn — xe chạy song song.
08. APIRouter dùng để làm gì?

APIRouter chia API thành module theo domain nghiệp vụ — mỗi domain 1 file riêng. Router có prefix URL, tags riêng cho docs, và có thể gắn dependency chung cho toàn bộ route trong router.

# routers/users.py
from fastapi import APIRouter, Depends

router = APIRouter(prefix="/users", tags=["users"])

@router.get("/{id}")
async def get_user(id: int): ...

@router.post("/")
async def create_user(): ...

# routers/transactions.py
router = APIRouter(prefix="/transactions", tags=["transactions"])

@router.post("/")
async def create_transaction(): ...

# main.py
from routers import users, transactions

app = FastAPI()
app.include_router(users.router)
app.include_router(
    transactions.router,
    dependencies=[Depends(require_auth)],  # auth cho toan bo /transactions
)
Như phòng ban trong công ty: CEO (app) chỉ quản lý trưởng phòng (router), không quản lý từng nhân viên.
09. Response Model dùng để làm gì?

Response Model khai báo cấu trúc dữ liệu trả về. FastAPI tự động lọc — chỉ giữ field có trong model, loại bỏ hết field không khai báo. Quan trọng cho bảo mật: endpoint có thể trả về object đầy đủ (kể cả password hash) nhưng chỉ field trong model mới ra đến client.

class UserInDB(Base):
    id: int
    email: str
    hashed_password: str   # KHONG muon expose

class UserResponse(BaseModel):
    id: int
    email: str             # chi 2 field nay duoc phep ra ngoai

@app.get("/users/{id}", response_model=UserResponse)
async def get_user(id: int, db=Depends(get_db)):
    return db.get(User, id)   # hashed_password tu dong bi loc

# response_model_exclude_unset=True -- bo field None khoi response
@app.patch("/users/{id}", response_model=UserResponse, response_model_exclude_unset=True)
async def update_user(id: int, data: UserUpdate): ...
Như tờ khai hải quan: dù trong kho có nhiều hàng, chỉ những gì trong tờ khai mới được qua cửa.
10. Làm sao implement JWT authentication trong FastAPI?

Flow gồm 3 bước: (1) /login xác thực user, trả JWT. (2) Các request tiếp theo gửi token trong header Authorization: Bearer <token>. (3) Dependency get_current_user tự động đọc, verify token và inject user vào endpoint.

from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm
import jwt

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login")

async def get_current_user(token: str = Depends(oauth2_scheme), db=Depends(get_db)):
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
        user_id = payload["sub"]
    except jwt.JWTError:
        raise HTTPException(status_code=401, detail="Invalid token")
    user = await db.get(User, user_id)
    if not user:
        raise HTTPException(status_code=401)
    return user

@app.post("/auth/login")
async def login(form: OAuth2PasswordRequestForm = Depends(), db=Depends(get_db)):
    user = await authenticate(form.username, form.password, db)
    if not user:
        raise HTTPException(status_code=401, detail="Wrong credentials")
    token = jwt.encode(
        {"sub": str(user.id), "exp": datetime.utcnow() + timedelta(hours=1)},
        SECRET_KEY, algorithm="HS256",
    )
    return {"access_token": token, "token_type": "bearer"}

@app.get("/me")
async def me(user = Depends(get_current_user)):   # tu dong verify
    return user
Như thẻ từ tòa nhà: quẹt thẻ ở cửa (đọc token), hệ thống check (verify), mở cửa và ghi nhận ai vào (inject user).
11. Làm sao handle CORS trong FastAPI?

CORS là cơ chế trình duyệt bảo vệ: khi frontend ở app.mybank.com gọi API ở api.mybank.com, browser hỏi server có cho phép không. FastAPI có CORSMiddleware tích hợp sẵn.

Điểm quan trọng: không dùng allow_origins=["*"] nếu API có xác thực — browser sẽ chặn việc gửi cookie/Authorization header kèm wildcard origin.

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://app.mybank.com",
        "https://admin.mybank.com",
    ],
    allow_credentials=True,          # cho phep gui cookie / Authorization header
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["Authorization", "Content-Type"],
)
Như danh sách khách được phép vào tòa nhà: bảo vệ kiểm tra, không có tên trong danh sách thì không vào.

13. Làm sao test FastAPI endpoint?

FastAPI có TestClient built-in — gọi endpoint trực tiếp không cần chạy server. Kết hợp dependency_overrides để thay DB thật bằng DB test.

from fastapi.testclient import TestClient

# Override dependency de test khong can DB that
def override_get_db():
    db = TestingSessionLocal()   # SQLite in-memory
    try:
        yield db
    finally:
        db.close()

app.dependency_overrides[get_db] = override_get_db
client = TestClient(app)

def test_create_user():
    response = client.post("/users", json={"email": "test@ex.com", "password": "abc"})
    assert response.status_code == 201
    assert response.json()["email"] == "test@ex.com"

def test_get_user_not_found():
    response = client.get("/users/999")
    assert response.status_code == 404

def test_unauthorized():
    response = client.get("/me")   # khong gui token
    assert response.status_code == 401
Như thử nghiệm ATM với tiền giả — kiểm tra mọi tình huống (rút thành công, quá số dư, sai PIN) mà không cần hệ thống thật.
14. Streaming response là gì? Khi nào dùng?

Thay vì chờ xử lý xong rồi gửi 1 lần, StreamingResponse gửi dữ liệu từng phần nhỏ ngay khi có. Dùng khi: export file CSV/Excel lớn, LLM trả về từng token, download file lớn. Lợi ích chính: RAM dùng gần như không đổi dù file có bao nhiêu dòng.

from fastapi.responses import StreamingResponse

async def generate_csv(user_id: int):
    yield "date,amount,description\n"      # header tra ve ngay
    async for row in db.stream(           # doc tung dong tu DB
        select(Transaction).where(Transaction.user_id == user_id)
    ):
        yield f"{row.date},{row.amount},{row.desc}\n"

@app.get("/export/transactions")
async def export_csv(user = Depends(get_current_user)):
    return StreamingResponse(
        generate_csv(user.id),
        media_type="text/csv",
        headers={"Content-Disposition": "attachment; filename=transactions.csv"},
    )

# LLM streaming -- tra tung token ngay khi co
async def stream_llm(prompt: str):
    async for chunk in llm.stream(prompt):
        yield f"data: {chunk}\n\n"   # Server-Sent Events format

@app.get("/chat")
async def chat(q: str):
    return StreamingResponse(stream_llm(q), media_type="text/event-stream")
Như truyền hình trực tiếp: xem từng khung hình ngay khi quay, không phải đợi quay xong cả buổi.
15. WebSocket trong FastAPI hoạt động thế nào?

HTTP: client gửi → server trả → kết thúc. WebSocket: kết nối duy trì hai chiều, cả hai bên gửi bất cứ lúc nào. Dùng cho: thông báo realtime giao dịch, cập nhật tỷ giá, dashboard live.

from fastapi import WebSocket, WebSocketDisconnect

class ConnectionManager:
    def __init__(self):
        self.connections: dict[int, WebSocket] = {}

    async def connect(self, user_id: int, ws: WebSocket):
        await ws.accept()
        self.connections[user_id] = ws

    async def send(self, user_id: int, data: dict):
        if ws := self.connections.get(user_id):
            await ws.send_json(data)

manager = ConnectionManager()

@app.websocket("/ws/notifications/{user_id}")
async def ws_notifications(websocket: WebSocket, user_id: int):
    await manager.connect(user_id, websocket)
    try:
        while True:
            data = await websocket.receive_text()
            await websocket.send_json({"echo": data})
    except WebSocketDisconnect:
        del manager.connections[user_id]

# Gui notification tu bat ky dau trong app
await manager.send(user_id, {"type": "transfer", "amount": 500000})
HTTP như gửi thư — gửi, chờ phản hồi, xong. WebSocket như gọi điện — kết nối duy trì, cả hai nói bất cứ lúc nào.
16. Làm sao implement rate limiting trong FastAPI?

Rate limiting giới hạn số request mỗi IP/user trong khoảng thời gian nhất định. Dùng Redis làm bộ đếm vì: rất nhanh và atomic — nhiều server cùng check không bị race condition.

# Cach 1: slowapi -- don gian nhat
from slowapi import Limiter
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter

@app.get("/api/rates")
@limiter.limit("60/minute")      # 60 request/phut/IP
async def get_rates(request: Request):
    return {"USD": 25000}

# Cach 2: Redis counter -- linh hoat hon, dung cho production
async def check_rate_limit(ip: str, limit: int = 100, window: int = 60):
    key = f"rate:{ip}"
    count = await redis.incr(key)
    if count == 1:
        await redis.expire(key, window)   # bat dau dem 60s
    if count > limit:
        raise HTTPException(
            status_code=429,
            detail="Too many requests",
            headers={"Retry-After": str(window)},
        )
Như quầy vé hạn chế số lượng: đếm người vào bằng Redis, quá ngưỡng thì từ chối đến hết giờ.
17. Làm sao trả về nhiều loại response từ một endpoint?

Khai báo tất cả response codes có thể trong responses dict để docs chính xác. Khi trả về response không phải default model, dùng JSONResponse trực tiếp.

from fastapi.responses import JSONResponse

class TransferSuccess(BaseModel):
    transaction_id: str
    amount: float

class TransferError(BaseModel):
    code: str
    message: str
    balance: float | None = None

@app.post(
    "/transactions",
    response_model=TransferSuccess,
    responses={
        200: {"model": TransferSuccess},
        400: {"model": TransferError, "description": "Loi business logic"},
        422: {"description": "Validation error -- sai format input"},
    },
)
async def transfer(data: TransferRequest, user=Depends(get_current_user)):
    if user.balance < data.amount:
        return JSONResponse(
            status_code=400,
            content={"code": "insufficient_funds",
                     "message": "So du khong du", "balance": user.balance},
        )
    result = await process_transfer(data, user)
    return result   # FastAPI tu apply response_model

19. Làm sao xử lý file upload trong FastAPI?

Dùng UploadFile nhận file qua multipart/form-data. Quan trọng: stream file thẳng lên S3 thay vì đọc hết vào RAM — đặc biệt khi file ảnh KYC có thể vài chục MB.

from fastapi import UploadFile, File, Form
import aioboto3

@app.post("/kyc/documents")
async def upload_document(
    file: UploadFile = File(...),
    doc_type: str = Form(...),
    user = Depends(get_current_user),
):
    ALLOWED = {"image/jpeg", "image/png", "application/pdf"}
    if file.content_type not in ALLOWED:
        raise HTTPException(400, "Chi chap nhan JPEG, PNG, PDF")

    # Doc tung chunk 64KB, kiem tra kich thuoc, khong load het vao RAM
    MAX_SIZE = 10 * 1024 * 1024   # 10 MB
    size = 0
    chunks = []
    while chunk := await file.read(64 * 1024):
        size += len(chunk)
        if size > MAX_SIZE:
            raise HTTPException(413, "File qua lon (toi da 10MB)")
        chunks.append(chunk)

    s3_key = f"kyc/{user.id}/{doc_type}/{uuid4()}{Path(file.filename).suffix}"
    async with aioboto3.Session().client("s3") as s3:
        await s3.put_object(Bucket=BUCKET, Key=s3_key, Body=b"".join(chunks))

    return {"key": s3_key, "size_bytes": size}
Như băng chuyền: hàng đi thẳng từ A sang B từng kiện, không cần kho chứa lớn ở giữa.
20. Làm sao implement health check endpoint đúng cách?

Phân biệt 2 loại: Liveness — app còn chạy không? Trả lời ngay, không check gì. Readiness — app sẵn sàng nhận traffic không? Check DB, Redis, external deps. Kubernetes dùng liveness để quyết định có restart không, dùng readiness để quyết định có route traffic vào không.

@app.get("/health/live", tags=["health"])
async def liveness():
    return {"status": "ok"}   # tra ngay, khong check gi them

@app.get("/health/ready", tags=["health"])
async def readiness():
    checks: dict[str, str] = {}

    try:
        await db.execute(text("SELECT 1"))
        checks["db"] = "ok"
    except Exception as e:
        checks["db"] = f"error: {e}"

    try:
        await redis.ping()
        checks["redis"] = "ok"
    except Exception as e:
        checks["redis"] = f"error: {e}"

    try:
        await payment_client.ping()
        checks["payment_gw"] = "ok"
    except Exception as e:
        checks["payment_gw"] = f"error: {e}"

    if any(v != "ok" for v in checks.values()):
        raise HTTPException(status_code=503, detail=checks)

    return checks

# Kubernetes config:
# livenessProbe:  GET /health/live  -> neu fail: restart pod
# readinessProbe: GET /health/ready -> neu fail: ngung route traffic, khong restart
Liveness: nhân viên còn trong văn phòng không? Readiness: nhân viên đã sẵn sàng nhận cuộc gọi chưa — dù ở văn phòng nhưng đang ăn trưa thì chưa ready.
