You

Yes, we can create AI interface first, this provide which method AI we can support. and if we can integrate with deepseek first. if another model AI added, we can switch to other model by config DB or environment variable

Project Manager

That's a good direction. I like that you're thinking about extensibility from the beginning.

I would refine the design a little.

Instead of having the rest of the application know about DeepSeek, OpenAI, or Claude, all business logic should only communicate with an AIProvider interface.

For example:

AIProvider chat() generate_lesson() evaluate_answer() correct_grammar()

Then we can have multiple implementations:

DeepSeekProvider OpenAIProvider GeminiProvider OllamaProvider

The application only depends on the interface. The actual provider is selected by configuration (environment variables) or a database setting.

That way, adding a new provider requires very little code.
